# Product Requirements Document (PRD)
**Product Name:** AriVision AI - Plant Pathology Diagnostic Lab  
**Document Version:** 1.0  
**Date:** March 2026  

---

## 1. Product Overview

### 1.1 Purpose
AriVision AI is a clinical-grade plant disease detection web application. It empowers farmers, botanists, and plant enthusiasts to accurately identify plant diseases, understand pathology, and receive actionable treatment and prevention protocols using advanced deep learning models.

### 1.2 Target Audience
*   **Farmers & Agriculturists:** Seeking rapid, reliable disease identification to protect crop yields.
*   **Home Gardeners:** Looking for easy-to-understand care tips and treatment options for household or garden plants.
*   **Botanists & Researchers:** Needing a fast preliminary diagnostic tool powered by modern AI architectures.

### 1.3 Value Proposition
By leveraging an ensemble of Vision Transformers (ViT) and Swin Transformers, AriVision AI provides highly accurate, pixel-level classification of plant pathology. The application delivers not just a diagnosis, but complete care instructions wrapped in a premium, highly responsive user interface.

---

## 2. Key Features

### 2.1 User Authentication System
*   **Sign Up & Log In:** Secure local authentication supporting email and password registration.
*   **User Sessions:** Persistent login state stored locally enabling users to return to their dashboard.
*   **Profile Management:** Autogenerated avatars and role-based access (admin/user).

### 2.2 Leaf Scanning & Image Input
*   **Camera Integration:** Direct access to device cameras to snap photos of leaves in real-time.
*   **Image Upload:** File browser integration for uploading existing `.jpg`, `.png`, or `.jpeg` files.
*   **Drag & Drop:** Interactive drop zones for seamless desktop uploads.

### 2.3 AI Diagnostics (Mocked/Simulation Layer)
*   **Ensemble Analysis:** Simulates analysis through Vision Transformer (ViT) and Swin Transformer pipelines.
*   **Detailed Pathology Report:** Returns:
    *   Target Plant & Leaf Variety
    *   Specific Disease Name
    *   Confidence Score (%)
    *   Condition Status (healthy, infected, critical)
*   **Actionable Advice:**
    *   **Cure/Treatment:** Specific chemical or organic fungicide recommendations.
    *   **Prevention:** Environmental control and crop rotation advice.
    *   **Ongoing Care:** Maintenance tips to prevent symptom recurrence.

### 2.4 User Dashboard & History
*   **Analytics Overview:** Displays total scans, overall plant health distribution, and recent activity.
*   **Scan History:** A historical log of previously scanned plants and their corresponding pathology reports.

### 2.5 Premium UI/UX 
*   **Glassmorphism Aesthetic:** Premium frosted-glass styled cards, dynamic gradients, and background blur.
*   **Ambient Animations:** Framer Motion-powered interactive elements, including ambient background visual effects like falling leaves.
*   **Responsive Design:** Fully mobile-optimized layout with adaptive navigation (Top Bar on Desktop, Bottom Nav on Mobile).

## 4. User Flow

1.  **Onboarding:** The user lands on the Home Page, viewing the product features and ambient animations.
2.  **Authentication:** The user clicks "Sign In" -> logs into an existing account or navigates to "Sign Up" to create a new profile.
3.  **Dashboard Access:** Upon login, the user views their Dashboard containing scan history and statistics.
4.  **Initiate Scan:** The user navigates to the "Scan Leaf" page.
5.  **Image Upload:** The user captures a photo with their camera or drops an image into the dropzone.
6.  **Analysis:** The user clicks "Analyze Pathology". An ambient scanning animation plays.
7.  **Results:** The system redirects to the Results Page, displaying the identified disease, confidence scores, and detailed care tabs.

---

## 5. Non-Functional Requirements
*   **Performance:** Page load times should remain under 2 seconds. The Vite dev server handles HMR instantly.
*   **Accessibility:** UI components follow WAI-ARIA standards via Radix UI.
*   **Scalability:** The frontend is decoupled from the AI inference engine, allowing easy scaling of real backend APIs in the future.
*   **Browser Compatibility:** Supports latest versions of Chrome, Firefox, Safari, and Edge.
